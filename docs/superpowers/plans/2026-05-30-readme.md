# Chatbot From Scratch

An end-to-end conversational NLP project that implements a decoder-only MiniGPT model from scratch, then compares it with pretrained and fine-tuned transformer baselines for dialogue generation.

This repository is intended as portfolio proof of practical AI/ML work: model implementation, data preparation, training, inference, evaluation, and automated testing in a single project.

## Why This Project Matters

- Demonstrates hands-on PyTorch model development beyond notebook-only experimentation.
- Shows practical NLP / generative AI workflow using dialogue data, tokenization, training, and inference.
- Includes both custom-model implementation and adaptation of pretrained transformer models.
- Adds engineering discipline through modular code, saved artifacts, and automated PyTest coverage.

## What I Built

- **Track A:** a pretrained conversational baseline loaded from Hugging Face (`DialoGPT`-style workflow).
- **Track B:** a custom decoder-only MiniGPT model implemented in PyTorch and trained on DailyDialog samples.
- **Track C:** a pretrained GPT-style model fine-tuned on the same dialogue task for comparison.
- A dialogue data pipeline covering formatting, tokenization, label masking, batching, and train/validation sampling.
- Inference and comparison scripts to inspect generated responses across different approaches.
- Automated tests for model components, data processing, checkpointing, and inference behavior.

## Technical Highlights

- Implemented token embeddings and learned positional embeddings with PyTorch layers.
- Built masked multi-head self-attention, feed-forward blocks, residual connections, and layer normalization for a decoder-only transformer.
- Used autoregressive next-token prediction with shifted logits / labels for causal language-model training.
- Applied label masking so training loss is computed on response tokens rather than prompt and padding tokens.
- Added checkpoint save/load support for the custom MiniGPT training workflow.

## Repository Structure

- `src/model.py` - custom MiniGPT architecture and autoregressive generation logic.
- `src/data.py` - DailyDialog loading, dialogue formatting, tokenization helpers, batching, and label masking.
- `src/train_track_b.py` - training loop and checkpointing for the custom MiniGPT model.
- `src/train_track_c_finetune.py` - fine-tuning workflow for a pretrained GPT-style baseline.
- `src/infer_track_b.py` - inference for the custom MiniGPT chatbot.
- `src/infer_track_c.py` - inference for the fine-tuned pretrained model.
- `src/compare.py` and `src/compare_track_c.py` - output comparison scripts across tracks.
- `tests/` - automated coverage for model, data, training utilities, and inference behavior.
- `artifacts/` - saved checkpoints, model artifacts, comparison outputs, and attention visualization.

## How to Run

Install dependencies:

```bash
python -m pip install -r requirements.txt
```

Run tests:

```bash
pytest -q
```

Train the custom MiniGPT model (Track B):

```bash
python -m src.train_track_b
```

Run inference with the custom MiniGPT model:

```bash
python -m src.infer_track_b
```

Fine-tune the pretrained GPT-style model (Track C):

```bash
python -m src.train_track_c_finetune
```

Run inference with the fine-tuned model:

```bash
python -m src.infer_track_c
```

Compare generated outputs:

```bash
python -m src.compare
python -m src.compare_track_c
```

## Evidence in This Repository

- Unit tests in `tests/` covering transformer components, data handling, checkpointing, and inference cleanup.
- A saved custom-model checkpoint in `artifacts/checkpoints/track_b_checkpoint.pt`.
- Fine-tuned model assets in `artifacts/track_c_distilgpt2/`.
- Sample comparison outputs in `artifacts/comparison.json` and `artifacts/comparison_track_c.json`.
- Attention visualization output in `artifacts/attention_heatmap.png`.

## Limitations

- This is an educational / portfolio project rather than a production-deployed system.
- The model comparisons are useful for inspection but do not constitute a rigorous benchmark study.
- Output quality depends on dataset subset size, hyperparameters, and generation settings.
- The repository focuses on modeling workflow and experimentation, not deployment infrastructure or agentic orchestration.
